get_next_line (1) 시작하기

서브젝트 이해하고 필요한 지식 공부하기

(아주 간단하게) 서브젝트 요약

Prototype

(2021년 7월 초에 프로토타입 변경됨)

Parameters

fd : 읽으려고 하는 파일 디스크립터

Return value

읽어들인 줄을 반환한다.
만약 아무것도 읽지 않았거나, 에러가 발생했다면 NULL을 반환한다.

Mandatory part

get_next_line 함수를 반복문으로 호출을 시키면, 파일이 끝나기 전까지 매 호출마다 한 줄을 반환한다.
표준입력에서 받아오는 것도 생각해줘야 함.
컴파일시에 -D BUFFER_SIZE=xx플래그를 사용해서 컴파일을 할 것이다. 컴파일 과정에서 read함수에서 사용할 버퍼 크기를 정해줄 것임.
static 변수를 사용하라는 힌트가 들어있음.

Bonus part

(난 하지 않았다.)

  • 한개의 static 변수만을 사용해서 get_next_line 구현하기
  • 여러개의 파일 디스크립터를 동시에 처리할 수 있도록 구현하기

    필요한 지식 공부하기

    -D Flag

    서브젝트를 보면 컴파일 할 때 gcc -Wall -Wextra -Werror -D BUFFER_SIZE=42 <files>.c. 이런 식으로 하겠다고 나와 있다.
  • D 플래그는 컴파일 시에 매크로를 정의하기 위한 플래그이고
  • D xxx=yy 이런 식으로 컴파일을 하면 #define xxx yy 를 소스코드에 ‘추가’한 것과 같다고 한다.
    처음에는 소스코드에 미리 define을 해 줘야 하는 줄 알고 (BUFFER_SIZE에 빨간 밑줄이 쳐져서 엄청 신경쓰였다) 살짝 헤맸는데 컴파일 시에 정의가 되는것이기 때문에 코드를 짤 때에는 오류가 있는 것이 맞아서 오류를 무시해도 잘 실행이 되었다.

    참고

    gcc 플래그에 대해서 정리해두신 블로그 글 링크

    Static 변수

    static으로 변수를 선언하면 컴파일시에 메모리가 할당되고, 할당된 메모리는 프로그램 종료시에 해제된다.
    static으로 선언한 변수는 선언된 소스 파일 내부에서만 접근할 수 있다.
    이 점에서 외부 파일에서도 접근할 수 있는 global 변수와 다르다고 할 수 있다.
    static 변수는 global 변수와 같이 메모리의 data section에 저장된다.
    참고로 초기화되지 않은 (초깃값이 없는) static 변수와 global 변수는 bss section에 저장된다고 한다.

    참고

    Wikipedia 링크
    코딩 도장 링크
    메모리 section에 대한 블로그 글 링크

    이 서브젝트에서의 필요성?

    파일의 한 줄의 길이보다 BUFFER_SIZE가 더 클 경우를 대비해서 static 변수가 필요하다. 파일 내용이 위와 같고 BUFFER_SIZE = 7이라고 하면
    첫번째 get_next_line 함수를 호출했을 때 반환은 “ABCDE”가 되겠지만 함수 내의 read 함수는 “ABCDE\nF”까지 읽었기 때문에 다음 줄을 읽기 위해 다음 get_next_line함수를 호출했을 때 다음 줄이 “FGHIJK”가 아닌 “GHIJK”가 된다.
    따라서 이전 호출에서 읽었던 F를 기억해놓아야 온전한 한 줄을 읽을 수 있고, 이를 위해서 함수의 호출이 끝나더라도 그 값이 유지되는 static 변수로 선언된 버퍼를 백업할 변수를 만들어놓아야 하는 것이다.
